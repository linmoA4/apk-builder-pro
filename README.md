# LM打包工具

这是一个面向手机端使用的 Android APK 工作台项目，不再只是项目骨架。当前版本已经把“启动准备环境、创建或导入工程、编辑文件、预检查、真实构建”串成了一条完整流程。

## 导入方式

1. 作为 Gradle 项目导入 Android Studio。
2. 或在 AIDE 中按目录打开，再按需要调整 Gradle 兼容版本。

## 当前特性

- 首次启动自动进入启动页
- 首次启动自动解压内置 `JDK 21`、`NDK r27c`、`Gradle 8.7`
- 二次启动先检查环境，正常后直接进入主界面
- 创建空壳 Android 工程，并真实写入 Gradle、Manifest、Java、布局和资源文件
- 导入 ZIP 或已有工程目录，并在同名冲突时自动改名保护旧目录
- 导入 ZIP 时显示真实解压进度
- 使用自定义进度卡片和自绘渐变进度条，而不是安卓原生 `ProgressDialog`
- JDK / NDK 多版本环境状态记录与切换
- 打包前只读环境检查
- 包名识别支持 `namespace`、`applicationId`、Manifest 多来源解析
- 构建时优先尝试内置离线 Gradle：`assets/toolchains/gradle/gradle-8.7-bin.zip`
- 对导入的带 `gradlew` 项目尝试执行真实 `assembleDebug`
- 应用图标和启动页头像已统一接入品牌图
- 液态玻璃背景已做性能优化，减少卡顿

## 当前结构

项目核心模块已经拆分为几类：

- `build/`：构建调用与预检查
- `env/`：JDK / NDK / SDK / Gradle 环境管理与安装
- `project/`：项目创建、导入、ZIP 解压、配置写回
- `service/`：工作流编排、工作区管理、文件树服务
- `theme/`：主题、液态玻璃背景、自定义进度条
- `model/`：环境状态、构建结果、项目配置等数据结构

## 启动流程

当前版本的启动逻辑是：

1. 进入 `StartupActivity`
2. 检查内置环境是否已准备
3. 如果缺失，则先解压内置 `JDK`、`NDK`、`Gradle`
4. 解压完成后再做一次环境校验
5. 校验通过后进入 `MainActivity`

这样做的目的是把环境准备前置，避免用户一进主界面就发现无法构建。

## 界面更新

最近几轮更新主要补了这几块体验：

- 启动页中间头像支持缩放动画
- 应用图标和启动页头像统一使用品牌图
- ZIP 导入时有真实百分比进度
- 旧的原生弹窗进度已替换成悬浮卡片 + 自定义玻璃进度条
- 液态玻璃背景去掉了最吃性能的软模糊，并降低了重绘负担

## 注意

- 真实打包依赖目标设备或环境里可用的 JDK、NDK、Android SDK、Gradle 运行条件。
- 预检查不再自动改写项目文件，缺少 `local.properties` 时会给出修复提示。
- 如果仓库内提供 `app/src/main/assets/toolchains/gradle/gradle-8.7-bin.zip`，构建时会优先解压并使用本地 Gradle，再回退到项目 Wrapper。
- 如果你要把它做成完全自动化、可长期维护的移动端开发工具，还可以继续加强 SDK 管理、图标资源方案、失败恢复和完整构建验证。
