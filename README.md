# APK Builder Pro 当前功能清单

这份文档按“现在这个项目已经能做什么”来整理，方便快速看全貌。

## 项目定位

`apk-builder-pro` 现在已经不是单纯的 Android 工程骨架，而是一个偏移动端使用场景的 APK 工作台。  
它把“项目创建 / 项目导入 / 文件编辑 / 环境准备 / 预检查 / 构建 / AI 辅助”串成了一条完整流程。

## 已有核心功能

### 1. 启动与主界面

- 首次启动会进入启动页
- 启动阶段走轻量检查，不再自动解压 APK 内嵌工具链
- 主界面分为首页和编辑页两块
- 整体 UI 使用液态玻璃风格、自定义卡片和自绘进度条

### 2. 项目创建

- 可以创建一个可直接打包的 Android 空壳工程
- 创建时会真实写入这些内容：
  - `Gradle` 脚本
  - `AndroidManifest.xml`
  - Java 源码
  - 布局文件
  - 资源文件
- 支持填写应用名、包名、版本号、版本代码
- 支持选择最低 SDK 和目标 SDK
- 支持给新项目设置应用图标和启动图

### 3. 项目导入

- 支持导入 ZIP 压缩包
- 支持导入已有工程目录
- 导入 ZIP 时会展示真实解压进度
- 导入时会自动识别 Android 工程根目录
- 同名项目导入时会自动改名，避免覆盖旧项目

### 4. 项目管理

- 首页可展示项目列表
- 长按项目可以打开管理面板
- 当前项目管理能力包括：
  - 打开项目
  - 重命名项目
  - 复制项目
  - 删除项目
  - 导出项目为 ZIP
  - 配置项目签名信息

### 5. 编辑器能力

- 支持项目文件树浏览
- 支持折叠 / 展开目录
- 支持在独立目录页浏览文件夹
- 支持多标签编辑
- 支持打开常见文本文件，例如：
  - `.java`
  - `.kt`
  - `.xml`
  - `.gradle`
  - `.kts`
  - `.properties`
  - `.md`
  - `.txt`
- 支持自动保存
- 支持撤销 / 重做
- 支持搜索与替换
- 支持错误行定位与高亮

### 6. 编辑器内置问题检测

- 对 XML 文件可做基础结构检查
- 对 Java / Kotlin / Gradle 文本可做括号平衡和字符串闭合检查
- 能识别合并冲突标记
- 检测到问题后会在编辑区显示建议卡片

### 7. 构建与打包

- 支持在项目内触发真实构建流程
- 打包前会执行预检查
- 会识别项目需要的：
  - JDK
  - NDK
  - Android SDK
  - Gradle 版本
- 对带 `gradlew` 的导入项目会尝试执行真实 `assembleDebug`
- 构建日志会实时写入日志区
- 打包成功后会显示 APK 输出路径
- 打包失败后会汇总错误并支持定位

### 8. 外置环境模式

- 项目已经改成“外置环境模式”
- 不再把 `JDK / NDK / SDK / Gradle` 大包内嵌进 APK
- 环境按项目需求动态检测和下载
- 支持多版本 JDK / NDK 状态记录和切换
- 支持国内镜像源 / 官方源两种下载路线

### 9. Wrapper 修复

- 会检查目标项目是否缺少 `gradle-wrapper.jar` 或相关配置
- 缺少时可以弹出 APP 自定义弹窗
- 支持一键补齐 Wrapper
- 支持两种补齐来源：
  - 仓库源
  - 官方源
- 补齐后可自动重新检测并继续构建

### 10. 签名配置

- 可以为项目配置 APK 签名
- 支持选择 `keystore`
- 支持填写：
  - `store password`
  - `key alias`
  - `key password`
- 保存后，打包流程可以直接使用已配置签名

## AI 相关功能

这个项目现在已经不只是“AI 聊天”，而是有几层不同能力。

### 1. AI 双引擎代码审查

- 可以对当前打开文件发起 AI 审查
- 目前是双模型并行审查
- 主要检查：
  - 语法错误
  - 资源引用错误
  - 明显空指针风险
  - Android API 调用问题
  - 逻辑漏洞
  - 边界条件问题
  - 状态同步问题
- 审查结果会进入错误列表，支持点击后直接跳到文件和行号

### 2. AI 情绪陪伴聊天

- 编辑页里有“陪我聊聊”
- 适合在 Bug 太多、打包失败时做陪伴式聊天
- 这一块偏情绪价值，不是执行型 AI

### 3. AI 后台执行器

这是这次新接入的重点能力。

- 新增“AI 写 APP”入口
- 可以直接输入一句目标，让 AI 开始改项目
- AI 在后台执行，用户看不到真实终端界面
- 但用户能看到 AI 的操作轨迹

当前这套执行器支持的可见能力包括：

- 展示 AI 的步骤摘要
- 展示 AI 执行了什么命令
- 展示命令输出
- 展示读取了哪个文件
- 展示读取了多少行
- 展示修改了哪个文件
- 展示文件 diff
- 新增内容用绿色显示
- 删除内容用红色显示
- 支持查看完整执行过程时间线
- 支持清空执行记录

注意：

- 读取文件时，界面上展示的是“读取哪个文件、读取多少行”
- 不会把文件正文原样直接暴露到这个过程面板里

### 4. 一键交给 AI 修错

- 错误列表弹窗已经改过
- 现在除了“提取全部错误”，还可以直接“一键交给 AI 修复”
- AI 会根据当前错误列表自动生成修复目标
- 然后直接进入后台执行流程

## 用户能直接感知到的交互特性

- 用户不用看真实终端
- 但能看到 AI 做了什么
- 能看到命令是否执行成功
- 能看到输出结果
- 能看到改了哪些文件
- 能看到新增和删除的内容差异
- 能看到 AI 的后台执行时间线

## 当前项目结构

核心代码大致分为这些模块：

- `build/`：构建调用与预检查
- `env/`：JDK / NDK / SDK / Gradle 环境管理与安装
- `project/`：项目创建、导入、ZIP 解压、配置写回
- `service/`：工作流编排、工作区管理、文件树服务
- `editor/`：编辑器、多标签、错误分析
- `ai/`：AI 聊天、AI 审查、AI 后台执行器、diff 渲染
- `theme/`：主题、液态玻璃背景、自定义进度条
- `model/`：环境状态、构建结果、AI 事件、项目配置等数据结构

## 目前最像成品的能力组合

如果只看“已经串起来的一条主线”，这个项目现在最完整的是下面这套流程：

1. 创建或导入 Android 项目  
2. 在编辑器里浏览和修改文件  
3. 用本地检测或 AI 审查找问题  
4. 直接一键把错误交给 AI 修  
5. 用外置环境准备 JDK / NDK / SDK / Gradle  
6. 打包前预检查  
7. 缺少 Wrapper 时一键补齐  
8. 继续执行构建

- 功能上已经具备“项目工作台 + 编辑器 + 构建器 + AI 辅助”的主体形态
- 新接入的 AI 后台执行器、过程展示和一键修错已经写进项目代码
